Copy pageCopy pageProjects on Vercel represent applications that you have deployed to the platform from a [single Git repository](/docs/git). Each project can have multiple deployments: a single production deployment and many pre-production deployments. A project groups [deployments](/docs/deployments)
and [custom domains](/docs/domains/add-a-domain).


While each project is only connected to a single, imported Git repository, you can have multiple projects connected to a single Git repository that includes many directories, which is particularly useful for [monorepo](/docs/monorepos) setups.


You can view all projects in your team's [Vercel dashboard](/dashboard), and selecting a project opens that project's dashboard, where you can:


li::marker]:text-gray-500">
* svg]:inline-flex last:mb-0">View an overview of the [production deployment](/docs/deployments) and any active pre-production deployments.

* svg]:inline-flex last:mb-0">Configure [project settings](/docs/project-configuration/project-settings) such as setting [custom domains](/docs/domains), [environment variables](/docs/environment-variables), [deployment protection](/docs/security/deployment-protection), and more.

* svg]:inline-flex last:mb-0">View details about each [deployment](/docs/deployments) for that project, such as the status, the commit that triggered the deployment, the deployment URL, and more.

* svg]:inline-flex last:mb-0">Manage [observability](/docs/observability) for that project, including [Web Analytics](/docs/analytics), [Speed Insights](/docs/speed-insights), and [Logs](/docs/observability/logs).

* svg]:inline-flex last:mb-0">Managing the project's [firewall](/docs/vercel-firewall).
